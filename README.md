
# mdbook-sspaeti

A custom mdBook preprocessor that adds WikiLinks and enhanced image handling.

## Features

### WikiLinks

Converts `[[term]]` to links pointing at a configurable base URL (e.g. a second brain). Slugs are generated by lowercasing, replacing spaces with hyphens, and stripping parentheses.

| You write | You get |
|-----------|---------|
| `[[Data Governance]]` | `[Data Governance](https://ssp.sh/brain/data-governance)` |
| `[[idempotency\|Idempotence]]` | `[Idempotence](https://ssp.sh/brain/idempotency)` |
| `[[MVC (Model View Controller)]]` | `[MVC (Model View Controller)](https://ssp.sh/brain/mvc-model-view-controller)` |

### Broken Brain Link Detection

When `brain-content-path` is configured, the preprocessor scans that directory at build time and builds a set of published slugs from `.md` file stems. WikiLinks whose slug is **not** found in the set are rendered as non-clickable broken links:

```html
<!-- Valid link (slug found) → normal markdown link -->
<a href="https://ssp.sh/brain/data-governance">Data Governance</a>

<!-- Broken link (slug not found) → no href, styled differently -->
<a class="brain-link broken">Minerva</a>
```

Broken links are logged to stderr with the source file for easy discovery:

```
mdbook-sspaeti: broken brain link: [[Minerva]] → minerva (part-2/6-dedp/dynamic-queries.md)
mdbook-sspaeti: broken brain link: [[Scuba]] → scuba (part-2/6-dedp/dynamic-queries.md)
```

If `brain-content-path` is not set, the preprocessor falls back to HTTP checking via `is-url-check` (slower, requires the brain site to be running). Links that fail the HTTP check are also rendered as broken.

### Image WikiLinks (Obsidian-style)

Embed images using Obsidian's `![[filename]]` syntax. Images are resolved from a configurable base path (default: `/images`).

```markdown
<!-- Simple image embed -->
![[roapi.webp]]

<!-- With caption -->
![[roapi.webp|ROAPI architecture overview]]

<!-- Absolute path (base path not prepended) -->
![[/images/subfolder/diagram.svg]]
```

Output for `![[roapi.webp|ROAPI architecture overview]]`:
```html
<figure>
<img src="/images/roapi.webp" alt="ROAPI architecture overview">
<figcaption>ROAPI architecture overview</figcaption>
</figure>
```

### Image Captions (rich markdown)

Place an italic line immediately after an image to create a `<figure>` with `<figcaption>`. The caption supports full inline markdown — links, bold, italic, code.

```markdown
![Data flow](/images/data-flow.webp)
_Image by [Staffbase](https://staffbase.com) about **Digital Workplace** Communication_
```

A blank line between image and caption is also fine:

```markdown
![Data flow](/images/data-flow.webp)

_This caption appears after a blank line and still works_
```

Output:
```html
<figure>
<img src="/images/data-flow.webp" alt="Data flow">
<figcaption>Image by <a href="https://staffbase.com">Staffbase</a> about <strong>Digital Workplace</strong> Communication</figcaption>
</figure>
```

### Image Sizing

Control image width using Pandoc-style attribute syntax:

```markdown
![alt](/images/small-diagram.webp){width=400px}
```

Combine with captions:

```markdown
![alt](/images/small-diagram.webp){width=400px}
_A smaller diagram with a caption_
```

Output:
```html
<figure>
<img src="/images/small-diagram.webp" alt="alt" style="max-width: 400px">
<figcaption>A smaller diagram with a caption</figcaption>
</figure>
```

### HTML Comment Stripping

HTML comments (`<!-- ... -->`) are automatically removed from chapter content so author notes and TODOs don't leak into the published HTML source. Comments inside fenced code blocks are preserved.

```markdown
<!-- TODO: rewrite this section -->   ← stripped from output

```html
<!-- this comment stays because it's in a code block -->
```
```

## Configuration

Add to your `book.toml`:

```toml
[preprocessor.sspaeti]
command = "mdbook-sspaeti"
brain-base-url = "https://ssp.sh/brain"   # base URL for [[wikilinks]]
is-url-check = false                       # HTTP-validate wikilink URLs on build (fallback, slow)
images-base-path = "/images"               # base path for ![[image]] embeds (default: /images)
brain-content-path = "/path/to/second-brain/content"  # detect broken wikilinks via filesystem
```

### Options

| Key | Default | Description |
|-----|---------|-------------|
| `brain-base-url` | `""` | Base URL for WikiLink targets |
| `is-url-check` | `false` | HTTP-validate each WikiLink on build (fallback when `brain-content-path` is not set) |
| `images-base-path` | `/images` | Directory prepended to bare image filenames in `![[...]]` |
| `brain-content-path` | — | Absolute path to second-brain content directory (e.g. Hugo `content/`). When set, `.md` file stems are used to detect broken WikiLinks at build time without HTTP requests |

## Installation

```bash
cargo install --path .
# or
make build
```

## Recommended CSS

Add figure styling to your mdBook's custom CSS (e.g. `assets/css/customize.css`):

```css
figure {
    margin: 1.5em 0;
    text-align: center;
}
figure img {
    max-width: 100%;
    height: auto;
}
figcaption {
    font-size: 0.9em;
    color: var(--fg);
    opacity: 0.6;
    margin-top: 0.5em;
    font-style: italic;
}
```
